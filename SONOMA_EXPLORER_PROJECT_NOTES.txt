SONOMA EXPLORER — PROJECT NOTES
================================
Last updated: April 2026


PROJECT OVERVIEW
----------------
Personalized interactive Sonoma County map app built with Leaflet.js + React/Vite.
Curated spots: wineries, restaurants, farmstands, and artisan producers.
Includes an AI "Sonoma Chef" assistant and a companion Expo mobile app.
The user is a professional chef/curator. GPS pin accuracy, website link accuracy,
and description accuracy are all critical.

GitHub (primary): ChefPlex/sonoma-explorer

                        ChefPlex/eldorado-county-explorer, ChefPlex/monterey-county-explorer

GPS COORDINATE VERIFICATION PROTOCOL
--------------------------------------
ALWAYS use this three-step process before committing any pin coordinate.

STEP 1 — OSM Nominatim (search by business name first):
  https://nominatim.openstreetmap.org/search?q=<Business+Name+City+CA>&format=json&limit=3

  If a POI result is returned (type: amenity, tourism, etc.), prefer it —
  it's the actual building footprint, not an address interpolation.

STEP 2 — Photon cross-check (corroborate Nominatim):
  https://photon.komoot.io/api/?q=<Business+Name>&limit=3

  Both sources should agree within ~50 metres.
  If they diverge, investigate further before committing.

STEP 3 — Fallback geocoders (use only if no POI exists in OSM/Photon):

  US Census TIGER geocoder (address-based, very accurate for US):
    https://geocoding.geo.census.gov/geocoder/locations/address
      ?street=<STREET>&city=<CITY>&state=CA&benchmark=2020&format=json

  Google Maps: extract coordinates from the URL using the !3d<LAT>!4d<LON> pattern.
  For restaurant/winery sites built on Squarespace, check the site's config JSON
  for embedded lat/lon values — often more precise than any geocoder.

RULE: Never use address geocoding alone as the sole source.
Always cross-check at least two independent sources.


CATEGORY SYSTEM
---------------
4 categories (stored as plain text in DB — no enum constraint):

  winery      Wine red      Tailwind 'primary'    Icon: wine glass
  restaurant  Amber         Tailwind 'secondary'  Icon: utensils
  farmstand   Sage green    #6f7d3c               Icon: leaf
  producer    Terracotta    #c06a2d               Icon: store

"producer" = artisan makers (creameries, cideries, spirits, etc.)
that don't fit cleanly into the other three categories.


DATABASE & SEED WORKFLOW
------------------------
- All new spot additions go into: artifacts/api-server/src/seed.ts
- Run the seed script after adding entries to populate the dev DB.
- Production DB is read-only. Production sync is triggered on deploy
  via correctCoordinates().
- Pins at the same physical property must be offset by ~0.0003°
  so they don't stack on the map.
  Example: Preston Farm & Winery has two pins —
    winery:    38.7030, -122.9674
    farmstand: 38.7033, -122.9671


MOBILE APP DETAILS
------------------
Bundle ID:   com.sonomachefapp.sonoma
Platform:    iOS (App Store) + Android
Framework:   Expo (React Native)
Config file: artifacts/sonoma-mobile/app.json

App Store version rules:
  - CFBundleShortVersionString must ALWAYS be higher than the last approved build.
  - Current version is set in app.json (check this before any EAS submission).
  - Build number is auto-incremented by EAS — do NOT hardcode it.
  - To ship a new version: run "eas build --platform ios" then "eas submit --platform ios".
  - Replit deployment republishes the existing binary — it does NOT trigger a new EAS build.
